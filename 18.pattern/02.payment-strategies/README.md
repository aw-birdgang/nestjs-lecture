
## Description
```bash

이 코드에서 PaymentController는 클라이언트 요청을 받아 해당 결제 방법에 맞는 전략을 설정하고, 결제를 진행합니다. 
PaymentService는 전략을 해결하고 설정하는 역할을 수행하며, 각 결제 방법에 대한 로직은 각각의 전략 클래스에서 정의됩니다.
이 구조를 사용하면 결제 방식의 추가나 변경이 필요할 때, 결제 서비스나 컨트롤러를 수정하지 않고도 새로운 전략 클래스를 추가함으로써 유연하게 대응할 수 있습니다. 
이는 Strategy 패턴의 주요 장점 중 하나로, 시스템의 확장성과 유지보수성을 향상시킵니다.

```
